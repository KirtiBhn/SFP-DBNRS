Introduction

Software Fault Prediction (SFP) models trained on class-imbalanced and overlapped datasets are prone to producing unreliable prediction models. In imbalanced datasets, traditional supervised learning algorithms typically favor the majority class, which impairs minority class classification and worsens performance due to prevalent class overlap issues. Numerous resampling techniques target class imbalance but ignore class overlap. In addition, most of the existing methods for SFP cannot handle the variable density of data effectively. We propose a Density-based Neighborhood Recursive Search (DBNRS) method to fill these gaps. The proposed method uses data densities of the minority classes to filter out excessively dense data from the majority instances in overlapping regions. The data densities are determined based on the neighborhood radius and the minimum number of neighbors within that radius, followed by the use of a neighborhood-based recursive search algorithm to enhance minority class visibility by removing majority class instances from overlapping regions. We eliminate the common nearest neighbors of two minority instances and apply the same approach recursively to the eliminated common neighbors. Additionally, four machine learning techniques and two ensemble learning methods are used to build SFP models from the pre-processed fault datasets. Extensive comparative findings on 13 datasets reveal that the proposed method provides the highest AUC score (= 81.1%), the maximum Recall value (= 78.2%), and the highest G-mean value (= 79.3%) as compared to the cutting-edge techniques. The experimental results showed that the proposed method enhanced the performance of fault prediction and surpassed current cutting-edge techniques in tackling class imbalance and overlap. These findings have significant practical applications, offering improved fault prediction capabilities that can be directly utilized to enhance software quality and lower maintenance costs in real-world software development and quality assurance processes.

Dataset Collection

For the experiment/analysis, dataset is collected from the open source PROMISE repository which is publically available.

Proposed Method Implementation

  1. Install Visual Studio code and python for implementation of the proposed method.
  2. All the absolute paths of the program should be set according to the system used.
  3. Install sklearn, numpy, imblearn and pandas libraries.
  4. Import the DBNRS.py and Undersampling.py files and run.
  5. Results are stored in results folder.

Softwares Required

  1. Windows 11 operating system
  2. For statistical purpose, SPSS tool by IBM is required
